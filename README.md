REST API для напоминаний – Laravel

Описание

Проект на Laravel 11 предназначен для управления напоминаниями с использованием REST API.
Система поддерживает полный CRUD для напоминаний, управление статусами, валидацию входных данных и заполнение базы тестовыми данными.

Используемые технологии

PHP 8+
Laravel 11
MySQL
Eloquent ORM
Миграции, фабрики и сидеры
REST API

Структура сущностей

Статус (Status)

Сущность "Статус" имеет единственный атрибут — наименование.

Поля:

id – bigint, первичный ключ
name – string, наименование статуса
created_at – timestamp
updated_at – timestamp

Предусмотренные статусы:

Новый
Исполнен
Запланирован
Просрочен

Напоминание (Reminder)

Сущность "Напоминание" содержит следующие поля:

id – bigint, первичный ключ
short_description – string, краткое описание
description – text, полное описание
created_at – datetime, дата и время создания
execution_datetime– datetime, дата и время выполнения
status_id – bigint, ссылка на статус
updated_at – datetime

Связь между сущностями

Один статус может иметь несколько напоминаний.
Одно напоминание принадлежит одному статусу.

Миграции и тестовые данные

Для выполнения миграций и заполнения базы тестовыми данными выполните команду:

php artisan migrate --seed

Сидеры автоматически создают следующие статусы:

Новый
Исполнен
Запланирован
Просрочен

Также создаются тестовые записи напоминаний.

REST API

Создание напоминания

POST api/tasks

Пример запроса:

{
"short_description": "Позвонить клиенту",
"description": "Уточнить детали договора",
"execution_datetime": "2026-01-20 14:00:00",
"status_id": 1
}

Получение всех напоминаний

GET /api/tasks

Обновление напоминания

PUT /api/tasks/1

Пример запроса:

{
"short_description": "Позвонить заказчику",
"status_id": 2
}

Удаление напоминания

DELETE /api/tasks/1

Получение списка статусов

GET /api/statuses

Валидация данных

short_description – обязательное поле
execution_datetime– обязательная дата
status_id – должен существовать в таблице statuses
